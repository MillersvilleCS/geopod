Geopod is open source and freely available for download. To get started using Geopod, follow the steps below. Please keep in mind that Geopod still in the testing stage. If you encounter any problems, feel free to contact us.

Geopod is available for Windows and Linux platforms. Unfortunately, Geopod is not compatible with Mac at this time. This is because Apple does not currently provide a package for Java3D, which Geopod requires for 3D rendering. If the Java3D package is provided some time in the future, Geopod will likely become available for Mac.

Requirements
Please ensure your computer meets the following requirements before installing Geopod:

  • Windows (XP/Vista/7/8) or Linux platform
  • Minimum 2 GB memory
  • Minimum 1366 x 768 screen resolution

Step 1: Install IDV
Geopod is a plugin for the Integrated Data Viewer (IDV), so before you can use Geopod you must have IDV installed on your computer. IDV versions 3.0u2 and 3.1u1 are supported. Older or more recent versions may not be fully compatible with Geopod. You can visit the IDV website to learn more about installing IDV.

Step 2: Run IDV Once
If you are already an IDV user and have not just installed a new version of IDV, you may skip this step.

If you have just installed IDV, it is important that you run it once so that it can create the necessary .unidata folder in your home directory. The .unidata folder is a hidden folder, so you will not normally see it, but IDV needs to create it so it can store data needed for plugins like Geopod.

Step 3: Install Geopod

For Windows users:
Download the following install script: InstallGeopod (EXE) then right click and choose run as administrator. This will update the necessary parts of IDV with the files from this website and install the latest version of the Geopod plugin. Note that InstallGeopod is an .exe file so some antivirus software may generate warnings when you download it.

The install script will need to locate IDV’s .unidata folder in order to install Geopod. If the .unidata folder cannot be located automatically, you will be prompted to find manually it using the file chooser provided. The .unidata folder is typically in your home directory, that is, the folder with the name of your account that is located under Users.

If you encounter problems: If for some reason the install script does not work, you can manually replace the necessary files as follows:

  1. Locate the IDV program folder. This will typically be C:/Program Files/IDV_VersionNumber unless you chose a different folder when you installed IDV.
  2. Download runIDV.bat (Windows Batch File)and use it to replace the runIDV.bat file in the IDV program folder. Note that if your computer hides file extensions (which it does by default) there will appear to be several files called ‘runIDV’ in the IDV program folder. Be sure to replace the one which is of type Windows Batch File.
  3. Download local-visad.jar (Executable Jar File) and use it to replace the local-visad.jar file in the IDV program folder.
  4. Locate your .unidata folder. This is typically in your home directory, that is, the folder with the name of your account that is located under Users. If you cannot find the .unidata folder and have just installed IDV, make sure you have run IDV at least once.
  5. Download the latest Geopod Plugin (Executable Jar File) and place it in the .unidata/idv/DefaultIdv/plugins folder.

For system administrators: If you are installing Geopod in a classroom setting where Geopod users do not have the administrator privileges needed to run the install script, use the following alternative:

  1. Run the PatchIDV Script (EXE) as administrator on all computers where IDV is installed to replace the necessary IDV files.
  2. Have individual Geopod users run the Geopod Update Script (EXE), which does not require administrator privileges, to install the Gepod plugin to their local .unidata folder.

For Linux users:
Download and run the following script: InstallGeopod.sh (shell script). This will update the necessary parts of IDV with the files from this website, as well as install the Geopod plugin.

If you encounter problems: If you encounter problems with the script, you can install Geopod manually as follows:

  1. Locate the IDV program folder. This will typically be YourHomeDirectory/IDV_VersionNumber unless you chose a different location when you installed IDV.
  2. In the IDV program folder, locate the runIDV script file (not runIDV.bat) and open it in an editor. Make the appropriate edits as specified below, then save your changes.
    For IDV version 3.1 or later: uncomment (remove two leading #'s) the line that sets idv_maxpermgensize=128.
    For older versions of IDV: Locate the line that invokes the java JVM (starts with java ...). The line directly below this one will be commented out (have two #'s in front) and be nearly identical except that it contains the text -XX:MaxPermSize=128m. Uncomment (remove the two #'s in front of) this line and comment out (place two #'s in front of) the line above it.
  3. Download local-visad.jar (Executable Jar File) and use it to replace the local-visad.jar file in the IDV program folder.
  4. Download the latest Geopod Plugin (Executable Jar File) and place it in the ~/.unidata/idv/DefaultIdv/plugins folder. If this folder does not exist, make sure you have run IDV at least once after installing.

Running Geopod
Step-by step instructions for running Geopod from IDV are provided in the Geopod Quick Start Guide (PDF). For an overview of the interface and more detailed instructions on using individual features, refer to the Geopod Users Guide (PDF). If you have questions or encounter problems, please contact us.


Updating Geopod
Geopod is frequently updated to include new features and bug fixes, so it’s a good idea to always make sure you are using the latest version.

For Windows users:
To update to the latest version of Geopod, download and run the Geopod Update Script (EXE). The script will need to locate IDV’s .unidata folder in order to install Geopod. If the .unidata folder cannot be located automatically, you will be prompted to find manually it using the file chooser provided. The .unidata folder is typically in your home directory, that is, the folder with the name of your account that is located under Users.

If you encounter problems: If for some reason you have difficulty using the update script above, you can follow the instructions for Linux users below to manually replace your Geopod plugin with the latest version.

For Linux users:
Locate the .unidata folder in your home directory. Download the latest Geopod Plugin (Executable Jar File) and use it to replace the existing GeopodPlugin.jar file it in the .unidata/idv/DefaultIdv/plugins folder.
